Hey everyone, thanks for joining. I'd like to walk you through the architecture of the Semi-Autonomous Software Development Factory we've been building. 

The core technical problem we're solving here is context degradation across the software lifecycle. Typically, when a PM hands a PRD to an architect, who hands a spec to an engineer, who hands code to QA, there is a massive loss of fidelity. To solve this, we've replaced the human handoffs with a directed graph of LLM agents, rigidly governed by a central state machine.

Here's how the stack actually works:

1. The Control Plane (FastAPI + Motor/MongoDB)
This is not a giant Langchain wrapper where agents talk to each other and hallucinate. Instead, it's a strongly-typed FastAPI application that acts as a centralized orchestrator. The pipeline is enforced by a state machine that pushes documents through explicit gates (PRD -> Spec -> Code -> Tests). All persistence relies on MongoDB Atlas, chosen because its schemaless document structure natively accommodates unstructured JSON/Markdown artifacts generated by the agents, cleanly managed with Pydantic schemas. 

2. The Agent Swarm (NVIDIA NIM APIs)
Instead of forcing one massive model to do everything, we use specialized tiers accessed via NVIDIA NIM:
- Spec & Code Generation: We use `qwen3.5-122b-a10b` for heavy-lifting structuring and architectural logic.
- Test Automation: We handoff to `mistral-small` since its codestral lineage is perfect for rapid functional coverage.
- Orchestration/Parsing: `moonshotai/kimi-k2.5` handles workflow parsing. 
- Validation Circuit Breakers: `deepseek-v3.2` acts as our linter/QA gate.

3. The Run Loop (Async + WebSockets)
When a PRD is POSTed to the ingestion endpoint, a BackgroundTask is spawned. The orchestrator pulls the first Agent class (BaseAgent wrapper handling rate-limits and retries), injects the PRD context, and executes via the OpenAI SDK wrapper. As each node completes, the Control Plane records an ExecutionDocument and ArtifactDocument in Mongo in real-time, while simultaneously broadcasting state transitions over a WebSocket event bus to the UI frontend. 

The end result is an audit-trailed, highly deterministic execution of generative AI that doesn't just write functions, but produces a spec-traced codebase.
